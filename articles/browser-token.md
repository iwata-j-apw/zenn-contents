# Claude Code × ブラウザ操作ツールのToken効率まとめ（所感ベース）

## 🧠 前提

ブラウザ操作におけるtoken消費は主に以下で決まる：

- 入力サイズ（DOM / スクショ / ログ）
- 操作ステップ数
- モデルの推論量

👉 本質的には  
**「どれだけ無駄な情報をモデルに食わせるか」** が支配的

---

## 🧭 ツール別所感（Token効率視点）

---

### 🔥 Chrome DevTools MCP

**総評：低〜中コスト（優秀）**

#### 特徴
- DOMを構造化して取得
- a11yツリーで軽量化
- 要素単位で操作可能

#### Token傾向
- ⭐⭐⭐⭐☆（低め）

#### 理由
- 必要な情報のみ取得
- HTML全体を投げない
- 差分的なやり取りが可能

#### 弱点
- ステップ数が増えると積み上がる

---

### 🟡 Claude in Chrome（拡張）

**総評：高コスト寄り**

#### 特徴
- スクリーンショットベース
- 自然言語操作
- UI理解に依存

#### Token傾向
- ⭐⭐☆☆☆（高め）

#### 理由
- 画面全体を毎回理解
- 画像 or 大量テキストを処理
- 推論コストが高い

#### 強み
- UXが直感的
- ノーコード操作

---

### 🔵 Playwright MCP

**総評：中コスト**

#### 特徴
- Playwrightベース
- テスト向け
- MCP経由で操作

#### Token傾向
- ⭐⭐⭐☆☆（中）

#### 理由
- DevToolsより抽象度が高い
- MCP層のオーバーヘッドあり

---

### 🟢 Playwright / Puppeteer（直接）

**総評：最も低コスト**

#### 特徴
- Claudeはコード生成のみ
- 実行はローカル環境

#### Token傾向
- ⭐⭐⭐⭐⭐（最小）

#### 理由
- ブラウザ状態をモデルに渡さない
- 推論はコード生成のみ

#### 弱点
- インタラクティブ性が低い
- リトライ制御が必要

---

### 🟣 Bun Headless Browser

**総評：Playwrightと同等**

#### 特徴
- Puppeteer互換
- 高速実行（Bun）

#### Token傾向
- ⭐⭐⭐⭐⭐（低）

#### 理由
- Claudeはコード生成のみ
- 実行は外部

---

### 🧪 browser-use / agent系

**総評：高コスト（重い）**

#### 特徴
- 自律エージェント
- ページ理解＋意思決定

#### Token傾向
- ⭐☆☆☆☆（かなり高い）

#### 理由
- 思考ステップが長い
- コンテキストが肥大化
- 操作回数が増える

---

## 📊 比較まとめ

| ツール | Token効率 | 特徴 |
|---|---|---|
| DevTools MCP | ⭐⭐⭐⭐☆ | 構造化データで効率良い |
| Claude in Chrome | ⭐⭐☆☆☆ | UX良いが重い |
| Playwright MCP | ⭐⭐⭐☆☆ | 中間層あり |
| Playwright / Puppeteer | ⭐⭐⭐⭐⭐ | 最小コスト |
| Bun Headless | ⭐⭐⭐⭐⭐ | 同上 |
| browser-use系 | ⭐☆☆☆☆ | エージェントで重い |

---

## 🎯 本質的な結論

👉 **Token消費は「どこで知性を使うか」で決まる**

- Claude側で理解 → 高コスト
- ローカルコードで処理 → 低コスト

---

## 🧪 ベンチマーク設計案

### 測定パターン
- ログイン操作
- フォーム入力
- ページ遷移
- ボタン押下

### 測定指標
- token使用量
- ステップ数
- 成功率
- 実行時間

### 比較対象（おすすめ）
- DevTools MCP
- Claude in Chrome
- Playwright（コード生成）
- browser-use

---

## 🚀 まとめ

- 最も効率良い → Playwright / Bun（コード生成型）
- バランス最強 → DevTools MCP
- UX重視 → Claude in Chrome
- 将来性ありだが重い → agent系
